Stage 3 — Merging deterministic pipeline_stage: merging

workbook_merger.merge() deterministic

tools/workbook_merger.py

Combines per-statement and per-notes xlsx files in stable order (SOFP → SOPL → SOCI → SOCF → SOCIE; then CORP_INFO → ACC_POLICIES → LIST_OF_NOTES → ISSUED_CAPITAL → RELATED_PARTY) into filled.xlsx. Preserves formulas, styles, values — does not recalculate (later correction agents must skip cell.value.startswith("=")).

repo.mark_run_merged fires immediately so GET /api/runs/{id}/download/filled works even if a later phase crashes (gotcha #10).

If a per-statement merge fails

SSE event: error with type: "merge_failed" is emitted carrying the MergeResult.errors list — no silent swallow (gotcha #20).

Stage 4 — Cross-checking (initial) deterministic pipeline_stage: cross_checking

cross_checks.run_all() deterministic

cross_checks/framework.py

Runs the registry of numeric reconciliations and emits per-check progress so the Validator tab fills in live (not batched — gotcha #19).

  • SOFP balance — assets = liabilities + equity
  • SOPL → SOCIE profit rollup
  • SOCI → SOCIE total comprehensive income
  • SOCIE → SOFP equity
  • SOCF → SOFP cash & equivalents
  • SoRE → SOFP retained earnings (MPERS only)

Group filings run each check twice (Group columns, Company columns). Status: passed · failed · pending · not_applicable. Each check emits cross_check_start / cross_check_result; pass ends with cross_check_complete (phase: "initial").

Stage 8 — Done terminal pipeline_stage: done

completed

All face + notes agents succeeded, merge succeeded, all cross-checks passed (initial or after one correction pass).

completed_with_errors

Pipeline finished but some agents failed, some checks remain failed, or correction couldn't reconcile every imbalance. Filled workbook still downloadable.

aborted / failed

User Stop-All → partial merge attempted before terminal status (gotcha #10). Hard exception → _safe_mark_finished still lands a terminal row.

Telemetry written throughout the run

Per turn — every model-request + tool-call node is captured to run_agent_turns (schema v8): turn_index, node_kind, tool_names, prompt/completion token deltas, cost, duration. Deltas computed by subtracting the previous cumulative agent_run.usage(); duration and tool calls are exact, the prompt/completion split is best-effort (gotcha #6).

Per run — rollups (prompt_tokens, completion_tokens, turn_count, tool_call_count) backfilled into run_agents at completion from result.usage.

Verbatim transcripts live on disk, not the DB — {stmt}_conversation_trace.json served on demand via GET /api/runs/{id}/agents/{stmt}/trace (path-confined). Failed agents save a partial trace too.

Invariants that govern the orchestration

  • Terminal-status contract (#10) — every exit path (success, exception, CancelledError, disconnect) lands a terminal status; never left as running.
  • Stop-All partial merge (#10) — on cancel, per-statement _filled.xlsx files already on disk are merged into a partial filled.xlsx before marking aborted, and a partial_merge SSE event lists what was kept.
  • Iteration cap below pydantic-ai (#18) — MAX_AGENT_ITERATIONS = 40 stays under pydantic-ai's silent 50 default; do not raise.
  • Soft page hints (#13) — scout suggestions are advisory; agents may view any PDF page. No allowed_pages filter.
  • Abstract-row write-guard (#17) — fill_workbook refuses writes to section-header rows (dark-navy fill) on both MFRS and MPERS.
  • Notes-cell HTML (#16) — agent emits HTML; sanitiser writes to notes_cells; xlsx download overlays DB rows onto template at stream time.
