How the journey changed

Before, the agent typed values straight into Excel cells and the spreadsheet was the only record of what happened. Now, extraction produces structured facts in a database — the spreadsheet becomes an export of that data, and every value, total, and correction is inspectable and reconcilable in the UI.

enabled by  XBRL_CANONICAL_MODE=1

The run, step by step

Before Legacy

The workbook is the source of truth.

  1. Upload a PDF and start the runPick statements, level, standard.
  2. Agents write into Excel cellsEach agent calls fill_workbook and edits its own {stmt}_filled.xlsx directly.
  3. Workbooks are mergedPer-statement files combine into one filled.xlsx.
  4. Cross-checks run on the spreadsheetTotals/consistency validated by reading cells back.
  5. Correction edits the spreadsheetOn failures, a correction agent rewrites cells in the merged .xlsx — opaque, with no structured record of what changed or why.
  6. Download filled.xlsxThe spreadsheet is the only artifact.
What the user sees Agent tabs, a cross-check list, and a download button. No structured view of the extracted numbers, no way to inspect a subtotal's makeup, no record of unresolved disagreements. To audit a value you open Excel and trace formulas by hand.

Now Canonical

The database of facts is the source of truth; Excel is an export.

  1. Concept trees load at startupEvery face template (MFRS/MPERS × Company/Group) is imported once so the system knows each statement's concepts, parents, and totals.
  2. Upload a PDF and start the runSame upload — but the run now carries a DB identity for its facts.
  3. Agents write structured factsEach extracted value is projected into run_concept_facts against a known concept — the database, not the cell, is authoritative.
  4. Totals compute automaticallyA cascade rolls leaf values up into subtotals and totals; mismatches become tracked conflicts.
  5. Workbook is exported from the DBLeaves become literal values; itemised totals keep live Excel formulas; "aggregate-only" totals become annotated literals.
  6. Correction works on factsThe correction agent resolves conflicts in the tree (revise a leaf, mark aggregate-only, mark not-disclosed), then the workbook is re-exported & re-merged — so the file always matches the data.
  7. Review & reconcile in the Concepts pageBrowse the concept tree, see each value's status, and clear the reconciliation queue. Unresolved conflicts are flagged on the results screen.
  8. Download filled.xlsxGenerated from the facts — it matches exactly what you saw in the UI.
What the user sees A new Concepts tab (and per-run links) showing the full concept tree with values, statuses, and source/evidence; a reconciliation queue for conflicts; and a banner on the results screen — "N unresolved conflicts need review" — when the run needs attention.

What actually changed

AspectBeforeNow
Source of truth The .xlsx file — whatever is in the cells. A database of structured facts; the spreadsheet is an export of it.
Visibility of data Open Excel and read cells; no in-app view of the numbers. A Concepts tree shows every value, its status, and its evidence.
Subtotals & totals Whatever the agent left in the cell. Computed by a cascade; itemised totals stay live formulas in Excel.
Corrections Opaque cell rewrites in the merged workbook. Tracked resolutions on the concept tree, then re-exported to Excel.
Disagreements Invisible unless a cross-check happened to catch them. Surfaced as a reconciliation queue and a results-screen banner.
UI ↔ file consistency A late cell edit could diverge from anything you'd reviewed. The download is regenerated from the facts — they can't drift apart.
"Done" signal Completed, unless a cross-check failed. Open conflicts flip the run to completed with errors for review.

Drift → In sync

Rollout & safety. The new journey is gated behind the XBRL_CANONICAL_MODE flag. With it off, the product behaves exactly as before (legacy path), so it's a one env-var rollback. The legacy spreadsheet pipeline still exists untouched underneath — the canonical layer sits on top of it.
Status. The pipeline is fully wired and test-backed end-to-end (import → extract → cascade → export → correct → reconcile). What's still owed is validation against real PDFs across MFRS/MPERS, Company/Group, and SOCIE filings before relying on it in production.
